# 0.3 Review Report: CoreData Model & Seeding

## 1. 目标验证

- **CoreData 模型**: ✅ (`Tally.xcdatamodeld` 存在且字段定义完整)
- **实体字段**: ✅ (Bill, Category, RecurringTask 字段与任务描述一致)
- **SeedService**: ✅ (已实现，包含预置分类和系统"未分类")
- **集成测试**: ⚠️ (代码已编写，但 App 入口未触发)

## 2. 检查项分析

### 2.1 代码逻辑

- **模型定义**:
  - `Bill`, `Category`, `RecurringTask` 实体的属性类型和命名均符合规范。
  - `Category` 包含 `isSystem` 字段，且 `SeedService` 正确设置了它。
- **SeedService**:
  - 使用 KVC (`setValue(_:forKey:)`) 进行赋值。虽略显脆弱（依赖字符串 Key），但在 `NSManagedObject` 子类生成前是有效手段。
  - `seedIfNeeded` 检查了现有数据量，防止重复写入。
  - `PersistenceController` 在 `init` 和 `preview` 中都正确调用了 `SeedService`。

### 2.2 关键问题 (Showstopper)

- **CoreData 栈未初始化**:
  - `PersistenceController.shared` 是懒加载单例。
  - `TallyApp.swift` 中**没有任何代码访问** `PersistenceController.shared`。
  - **后果**: App 启动后，`loadPersistentStores` 不会被调用，数据库文件不会被创建，`SeedService` 也不会运行。验收标准无法通过。

### 2.3 改进建议

1.  **初始化 CoreData**: 在 `TallyApp.swift` 的 `init` 或 `AppEnvironment` 中显式初始化 `PersistenceController`。
    ```swift
    // 建议在 AppEnvironment 中持有，或在 App init 中调用
    struct AppEnvironment {
        let persistenceController = PersistenceController.shared // 强制触发 init
        let container: DIContainer
        ...
    }
    ```
2.  **文件移动**: 再次建议将 `Tally/Persistence.swift` 移动到 `Tally/Data/Persistence.swift`，保持架构整洁。

## 3. 结论

CoreData 模型与 SeedService 逻辑实现良好，但因**缺乏入口调用**导致功能未生效。需修复 App 启动时的初始化逻辑。
