# 17 导出功能 MVP（CSV + JSON）

## 目标

- 完成可用导出闭环：导出 CSV（分析用途）+ 导出 JSON（备份用途）。
- 文件可通过系统分享面板导出，用户可在“文件”App中查看与保存。
- 保证字段与命名规范稳定，后续导入可复用同一数据结构。

## 开发步骤

1. 导出 JSON（备份）
   - 导出内容至少包含：
     - 账单（Bill）
     - 分类（Category）
     - 定时记账（Recurring）
   - 增加元数据：
     - `schemaVersion`
     - `exportedAt`
     - `appVersion`
     - `timezone`
   - 文件名规范：
     - `Tally_Backup_YYYYMMDD-HHmmss.json`

2. 导出 CSV（账单）
   - 字段顺序固定为：
     - 时间、类型、分类、金额、备注
   - 编码使用 UTF-8（建议带 BOM，兼容 Excel 中文）。
   - 文件名规范：
     - `Bill_YYYYMMDD-YYYYMMDD.csv`

3. 范围与筛选（MVP）
   - 支持导出范围：
     - 当前月
     - 全部
   - 时间范围仅影响账单导出，分类/定时记账在 JSON 中按完整备份导出。

4. 文件输出与分享
   - 导出到临时目录后，通过系统分享面板导出。
   - 导出完成反馈：
     - 文件类型
     - 记录条数
     - 文件大小（可选）

5. 数据格式细节
   - 金额导出为字符串，固定两位小数。
   - 时间建议导出 ISO8601（UTC），并保留 `occurredLocalDate` 用于回放分组。

## 验收标准

- ✅ 能成功导出 JSON 备份文件并分享。
- ✅ 能成功导出 CSV 文件并分享，字段顺序正确。
- ✅ 文件命名符合规范。
- ✅ 导出后可读性正常（CSV 打开不乱码，JSON 可解析）。
- ✅ 导出过程不阻塞 UI（大数据量时不明显卡顿）。
