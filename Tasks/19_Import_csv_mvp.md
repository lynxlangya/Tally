# 19 导入 CSV（账单）MVP

## 目标

- 支持从 CSV 导入账单数据（以账单为主，不导入分类和定时记账）。
- 在保证容错与可解释错误的前提下，完成基础迁移能力。
- 导入后账单口径符合现有业务规则（金额、分类、日期、时区）。

## 开发步骤

1. CSV 解析器
   - 解析固定列：
     - 时间、类型、分类、金额、备注
   - 支持 UTF-8（含 BOM）输入。
   - 列名不匹配时中止并提示模板格式。

2. 行级校验
   - 时间可解析（建议支持 ISO8601 与 `yyyy-MM-dd HH:mm:ss`）。
   - 类型仅允许：
     - 收入 / 支出
   - 金额必须 > 0，最多两位小数。
   - 分类为空时回退「未分类」。

3. 分类映射策略
   - 按「类型 + 分类名」优先匹配已有分类。
   - 匹配不到时（MVP 二选一，固定一种并文档化）：
     - 方案 A：自动创建新分类（推荐）
     - 方案 B：回退未分类

4. 重复判定与处理
   - 默认策略：跳过重复。
   - 重复判定使用现有规则：
     - 同金额 + 同分类 + 同 occurredLocalDate

5. 结果报告
   - 输出：
     - 成功数
     - 跳过数
     - 失败数
   - 提供失败明细（行号 + 原因）。

## 验收标准

- ✅ 可选择 CSV 文件并完成解析。
- ✅ 合法行可导入，非法行有明确报错且不影响其他行。
- ✅ 重复数据按规则被跳过，不会重复入库。
- ✅ 导入后首页与明细统计口径正常。
- ✅ 中文内容（分类名/备注）导入后无乱码。

## 当前实现记录（MVP）

- 已接入完整流程：`选择 CSV 文件 → 预检统计 → 用户确认 → 执行导入 → 结果弹窗`。
- 列头严格校验：仅接受 `时间、类型、分类、金额、备注`。
- 编码处理：支持 UTF-8（含 BOM）。
- 时间解析支持：
  - ISO8601（含毫秒）
  - `yyyy-MM-dd HH:mm:ss`
- 类型支持：`收入/支出`、`income/expense`。
- 冲突判定已落地：同 `金额 + 分类 + occurredLocalDate` 视为重复并跳过。
- 分类映射策略（本期固定）：
  - 按 `类型 + 分类名` 匹配现有分类
  - 未匹配到时回退系统「未分类」
  - 本期不自动创建新分类（降低脏数据风险）
- 预检错误摘要：展示前 3 条 `行号 + 原因`，用于快速修复源 CSV。
