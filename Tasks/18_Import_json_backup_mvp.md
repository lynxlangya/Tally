# 18 导入备份（JSON）MVP

## 目标

- 实现 JSON 备份导入闭环：选择文件 → 预检 → 确认导入 → 输出结果。
- 导入优先保证“可恢复性”和“数据一致性”，不追求复杂映射。
- 导入过程支持失败可回滚，避免脏数据写入。

## 开发步骤

1. 文件选择与解析
   - 使用系统文档选择器选择 `.json` 文件。
   - 校验必需元数据：
     - `schemaVersion`
     - `exportedAt`
   - 不兼容版本给出明确提示并中止。

2. Dry-Run 预检（不写库）
   - 统计：
     - 可导入条数
     - 冲突条数
     - 失败条数
   - 展示错误摘要（前 3 类）：
     - 字段缺失
     - 金额非法
     - 类型非法

3. 导入执行（事务）
   - 用户确认后开始写库。
   - 采用事务/批次写入策略，发生结构性错误时整批回滚。
   - 导入顺序建议：
     1) 分类
     2) 账单
     3) 定时记账

4. 冲突策略（MVP）
   - 分类同 ID：更新已有记录（保护系统分类关键字段）。
   - 账单同 ID：跳过（避免重复覆盖）。
   - 关联分类缺失：回退到系统「未分类」。

5. 导入结果反馈
   - 展示：
     - 成功 N
     - 跳过 N
     - 失败 N
   - 提供失败摘要文本（可复制）。

## 验收标准

- ✅ 可从系统文件中选择 JSON 并完成预检。
- ✅ 点击确认后可完成导入，结果统计正确。
- ✅ 导入失败不会造成半写入脏数据。
- ✅ 导入完成后首页/明细/类别页面数据能正确刷新。
- ✅ 未分类回退逻辑有效，不会出现空分类引用。
